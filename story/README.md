# story

故事初始化资料目录。

导入人物卡、故事书、世界书时，程序会在这里为每个导入文件建立一个子目录：

- 保留原始文件，例如 `.json`、`.png`、`.md`、`.txt`。
- 生成一个转换后的 Markdown 文件，供人直接阅读和后续维护。
- 生成 `manifest.json`，记录条目、人物和来源文件。
- 生成 `program-config.json` 和 `program-config.md`，把来源格式不统一的人物卡、故事书、世界书加工成程序需要的固定结构。

这里保存的是故事开局和设定资料，不保存游玩进度。

`program-config` 包含世界观、登场人物介绍、第一轮开场交互、初始玩家选项、人物状态 schema、初始人物状态、规范化资料条目和初始全局上下文。初始化阶段不生成长期大纲和剧情线。

生成 `program-config` 时会调用一次 LLM；必须在前端 `故事库` 手动点击初始化。已有 `program-config` 时可直接复用，不重复生成。初始化 prompt 在 `prompts/initializer/prompt.md`。

网页“开始新游戏”只列出已初始化故事。未初始化、初始化失败、缺少 API Key 或配置字段不完整时不能开始游戏。
