# History Compact

把两组历史记录分别压缩成各 1 条历史总结。

## 规则

- 只保留后续会影响判断的因果、人物关系、未解线索、状态变化和剧情方向。
- 删除重复环境描写、低价值动作、临时氛围、重复播报和无后续影响的细节。
- 不写未来计划，不新增事实，不改写已发生事实。
- 每条压缩结果尽量短，但必须保留关键因果。

JSON 形状：

```json
{
  "compactA": "第1组压缩后的单条历史总结",
  "compactB": "第2组压缩后的单条历史总结"
}
```
