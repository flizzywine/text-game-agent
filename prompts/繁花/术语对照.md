# 繁花术语对照

| 基准术语 | 繁花术语 | 含义 |
|---|---|---|
| 玩家 | 顾客 | 发出需求的人，输入来源 |
| 玩家输入 | 顾客需求 / 顾客言语 | 本轮用户输入 |
| 玩家负反馈 | 顾客留给花神的话 | 用户手动保留的持续要求 |
| 世界观 | 下面是花朵占卜出与顾客有命运纠缠的世界观信息 | 故事设定和规则 |
| 剧情目标 | 花朵的预言方向 / 近期预言走向 | 未来若干轮要靠近的具体事件目标 |
| 当前剧情目标 | 当前花语 / 当前预言走向 | 当前长期目标 |
| 长期总结 / 历史总结 | 已显现的花语记录 / 过往预言记录 | 已发生事实的压缩记忆 |
| 最近正文 | 近期预言正文 / 近几次花语显现 | 反重复、承接用上下文 |
| 当前人物状态 | 花色观察 | 记忆锚点，不是建模控制 |
| 导演 | Hana / 花语译者 / 预言编排者 | 把顾客需求转成结构化计划 |
| Director 计划 | 花语计划 / 本轮预言结构 | `beat1/beat2/beat3/ending/physicalConstraints` |
| Narrator | 花语书写者 / 预言正文书写者 | 把计划写成正文 |
| Postprocess | 花语校验 | 总结、状态补丁、花神的嘱咐 |
| HighLevelDirector | 花语长期走向判定者 | 生成或更新长期预言方向 |
| 剧情推进 | 预言推进 / 花色变化 | 让故事向前流动 |
| 扰动 | 花色变化 / 花语异动 | 新信息、新阻力、新误会等 |
| 节拍 beat | 花色层次 / 预言片段 | 三段计划结构 |
| 结尾窗口 | 花语停留处 / 顾客可回应的入口 | 留给用户下一轮行动 |
| 物理约束 | 花语中的现实限制 / 预言不可违背的物理条件 | 纯物理禁止事项 |
| 负反馈 / 闭环 | 花神的嘱咐 | 下一轮必须遵守、修正或避免的内容 |
| 玩家选项 | 顾客命运分叉 | Hana 猜测顾客接下来可能走向的三条分叉 |
| 正文 | 预言故事 / 花朵显现的故事 | 用户可见小说正文 |
| 故事叙事风格 | 花朵呈现的文风 / 花色文风 | 文体要求 |
| 导演风格 | 花语编排方式 / 预言推进风格 | 冲突、节奏、关系变化策略 |

## 替换原则

- 只替换 `prompts/繁花/` 的 prompt 文案。
- 不改程序变量名，例如 `playerInput`、`longRangeOutline`、`directorPlan`。
- 不改 JSON key，例如 `beat1`、`draftText`、`playerOptions`。
- 目标是测试术语壳对模型行为的影响，并保留 `prompts/基准/` 可回退。
