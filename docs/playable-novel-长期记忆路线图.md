# Playable Novel 长期记忆路线图

目标：从“即时互动叙事引擎”升级为“可导入长篇小说、支持任意位置开局、引导玩家完整走完原著剧情的 playable novel 引擎”。

当前阶段不处理玩家突破原著、自由分支世界线、多 agent 自主推演。先验证核心机制：长期记忆、长期连贯、长期文本一致性不能崩。

## 核心原则

- 原著是主轨道，玩家默认被引导沿原著推进。
- 摘要只做导航，原文才是事实证据。
- 每轮必须知道当前处于原著哪里。
- 人物不能靠常驻上下文硬塞，必须能按场景、关系、关键词召回。
- Director 负责贴近原著轨道，Narrator 只写当前正文。
- 先做固定流程，不做自由 agent。

## 阶段 1：原著导入层

目标：把长篇小说变成可定位、可检索、可追踪的数据。

产物：

| 模块 | 内容 |
| --- | --- |
| 章节切分 | `chapter_id`、标题、正文 |
| 场景切分 | `scene_id`、所属章节、地点、人物、时间、原文 |
| L0 原文层 | 不改写的原始正文切块 |
| L1 场景摘要 | 每个场景发生什么、谁在场、冲突是什么 |
| L2 章节摘要 | 本章推进了什么剧情、人物关系有什么变化 |
| L3 全书主线 | 长期剧情阶段、主线事件、人物弧线 |

完成标准：

```text
能从任意章节/场景定位原文
每个摘要能反查到 L0 原文
系统不会只靠模型自由发挥
```

## 阶段 2：人物长期记忆

目标：长篇人物众多时，系统只召回当前需要的人物，不把所有人物塞进上下文。

产物：

| 模块 | 内容 |
| --- | --- |
| 人物表 | 姓名、别名、身份、阵营、初登场位置 |
| 出场索引 | 每个场景实际出现了谁 |
| 提及索引 | 谁被谈到但没有出场 |
| 人物弧线 | 每个阶段的人物状态、目标、关系变化 |
| 当前人物池 | 当前场景允许出现、可被提及、禁止乱入的人物 |

人物召回输入：

```text
当前 source_anchor
当前地点
当前场景人物
玩家输入中的人物名/别名
原著下一事件需要的人物
```

人物召回输出：

```text
核心在场人物
可选登场人物
只可提及人物
不应出现人物
人物当前阶段档案
相关原著片段
```

完成标准：

```text
当前场景知道谁应该在场
配角不会因为不在上下文里消失
无关人物不会突然乱入
人物关系随原著阶段变化
```

## 阶段 3：原著轨道控制

目标：先不允许大分支，Director 每轮负责把玩家行动带回原著附近。

每轮 Director 输入：

```text
当前 source_anchor
当前场景 L1 摘要
上一场景 L1 摘要
下一原著事件
当前人物池
必须推进的剧情点
禁止提前泄露的信息
最近玩家正文
```

Director 输出：

```text
本轮剧情目标
允许出现的人物
必须保持一致的事实
本轮应靠近的原著事件
下一候选 source_anchor
```

完成标准：

```text
玩家普通互动不会带偏主线
每轮都向原著下一事件靠近
剧情不会跳章、漏章、提前揭示
```

## 阶段 4：多路召回

目标：每轮只取当前需要的信息，避免长篇上下文爆炸。

固定流程：

```text
current_anchor
  -> 剧情召回
  -> 人物召回
  -> 玩家历史召回
  -> Director
  -> Narrator
  -> OptionStrategist
  -> Summary / 状态更新
```

三路召回：

| 召回 | 内容 |
| --- | --- |
| 剧情召回 | 当前场景附近的 L1/L0、上一场景、下一事件 |
| 人物召回 | 当前人物池的人物档案、关系、原著阶段状态 |
| 玩家历史召回 | 玩家已经经历过的最近轮次和相关旧轮次 |

完成标准：

```text
连续游玩几十轮后仍知道当前原著位置
人物说话符合当前阶段
叙事不重复、不失忆、不自相矛盾
```

## 阶段 5：长程一致性检查

目标：每轮结束后更新状态，防止越跑越歪。

Postprocess 写入：

```text
已完成的原著事件
当前 source_anchor
当前人物状态
当前人物关系
本轮新增事实
下一候选场景
```

一致性检查：

```text
是否提前发生未来剧情
是否修改原著既定事实
人物是否出现在不合理地点
人物关系是否和当前阶段冲突
当前剧情是否偏离 source_anchor
```

完成标准：

```text
系统能跑完一章
系统能跑完连续三章
系统能从中途存档继续
系统能解释当前处于原著哪里
```

## 阶段 6：Playable Novel MVP

目标：做一个能验证长篇原著运行能力的最小版本。

必须做到：

```text
导入一部长篇小说
选择任意章节/场景开局
系统引导玩家走完该章
人物登场合理
剧情沿原著推进
每轮可追踪 source_anchor
存档后可继续
```

暂不做：

```text
玩家突破原著
自由分支世界线
复杂人物自主行动
多 agent 并行推演
全书级开放世界
```

## 验收顺序

1. 单章跑通：从一个章节开局，完整走完该章。
2. 三章连续：从第一章跑到第三章，人物和剧情不断线。
3. 中途开局：从任意场景开局，系统能正确补足当前位置。
4. 中途存档恢复：保存后继续，`source_anchor`、人物状态、已完成事件不丢。
5. 长轮次压力测试：连续几十轮后，仍能说明当前处于原著哪里、下一原著事件是什么。

## v2.0 定义

v2.0 的核心不是自由度，而是长篇原著可运行。

判断标准：

```text
系统能稳定召回原著
系统能稳定召回人物
系统能维持文本一致性
系统能引导玩家完整走完原著剧情
```
