# Text Game Agent

Text Game Agent 是一个本地文字游戏 agent，用来把故事资料、人物状态和玩家选择持续转化成可游玩的小说正文。

它不是酒馆式 chatbox，也不是一套更长的提示词预设。它把互动小说拆成多次、有分工的模型调用：先判断剧情该怎么推进，再写正文，再给出下一轮剧情方向，同时由程序维护长期状态和历史压缩。

## 相比酒馆的优势

酒馆的核心模式是单次长提示词：角色卡、世界书、历史记录、规则、风格、禁忌和玩家输入都会被塞进同一次模型调用里，让模型一次性完成理解、规划、写作、记忆和自我修正。

Text Game Agent 的核心模式是多阶段文字游戏 agent。程序负责拆任务、保存状态、压缩历史、调度模型；模型每次只处理一个相对明确的任务。

### 1. 任何故事材料都能变成可玩的游戏

酒馆通常依赖角色卡、世界书和预设格式。材料不规范时，用户需要自己整理设定、拆人物、写开场、补世界观。

Text Game Agent 可以直接导入纯文本小说、酒馆人物卡、世界书、零散设定文件，程序会调用 LLM 做初始化：整理世界观、人物状态、开场正文和初始剧情方向。

对用户来说，直接收益是：不用先把资料整理成固定格式，也不用从零写角色卡；小说、人物卡、世界书或零散设定都可以作为游戏起点。

### 2. 可以随心所欲修改已有故事

酒馆更像是在已有角色卡和世界书上继续聊天，用户想大幅修改人物关系、世界规则、开局状态或剧情方向时，很容易和原设定互相打架。

Text Game Agent 会把故事资料初始化成可编辑的 program config。用户不必自己手写配置，可以直接用自然语言指导 AI 修改人物设定、世界设定、关系结构、开局配置和剧情方向，再用修改后的版本开始游戏。

对用户来说，直接收益是：可以把别人的小说、角色卡或世界书当成原料，而不是必须照单全收；想改人设、改关系、改世界观、改开局，都可以改成更符合自己心意的可玩世界。

### 3. 弱模型也能产出更稳定的高质量正文

酒馆式长提示词对模型能力要求很高。模型必须一次性理解设定、回忆历史、规划剧情、写正文、避免重复、更新状态。弱一点的模型很容易顾此失彼：文笔变水、设定遗忘、人物漂移、剧情原地打转。

Text Game Agent 把这些压力拆开。每一步只让模型完成一个更明确的任务：先决定剧情怎么走，再写正文，再规划下一轮方向，后台再整理记忆。

对用户来说，直接收益是：不一定非要上最强、最贵的模型，也能得到更稳定的剧情推进、更少失忆、更少重复、更接近强模型的整体输出质量。

### 4. 文字表达不容易反复

酒馆式文游常见的问题是越玩越像同一套句子在循环：反复写眼神、沉默、呼吸、手指、空气、暧昧气氛，同一种动作和心理描写一轮接一轮出现。

Text Game Agent 会在正文生成前检查本轮是否又在重复旧措辞、旧动作描写、旧感官词、旧心理解释和同类句式，把这些问题提前压住。

对用户来说，直接收益是：正文更少水词、套话和重复描写，读起来不像同一批句子反复换位置，人物对话和动作也更不容易机械复读。

### 5. 角色不容易失忆和性格漂移

酒馆主要靠上下文里的文本记忆维持一致性。玩得久了，角色容易忘记之前发生过什么，关系变化容易被冲淡，人物性格、位置、身体状态和世界规则也会逐渐漂移。

Text Game Agent 会把世界观、人物状态、人物性格、关键记忆、物理约束和关键信息保存为程序里的长期变量，而不是只依赖模型在长上下文里自己记住。

对用户来说，直接收益是：角色更记得自己是谁、和别人是什么关系、上一轮处在什么状态；长期关系变化、重要承诺、误会、伤势和位置不容易被后续正文冲掉。

### 6. 玩得久也能记住历史

酒馆式文游玩久之后，历史正文会越来越长。模型要么忘记很早之前发生过什么，要么被大量旧文本干扰，导致当前剧情失焦、人物关系漂移、前后因果接不上。

Text Game Agent 会把历史正文压缩成分层记忆：近期正文保留细节，较早剧情保存成事实总结，更长历史继续分段压缩。需要旧细节时，再回看相关旧轮次。

对用户来说，直接收益是：玩很多轮以后，系统仍然能记得过去的重要事件、关系变化、承诺、误会和伏笔，不容易因为上下文过长而失忆或漂移。

### 7. 剧情节奏更快，不容易拖沓

传统文游选项常常是即时动作或台词，例如“继续问她”“走过去”“沉默等待”。这种选项会让剧情一直贴着当前场景滑行，人物反复寒暄、试探、等待反应，节奏越来越慢。

Text Game Agent 的候选项不是角色台词或动作，而是未来剧情方向。选项策略器从更高层级生成 5 个方向，类型包括 `推进`、`转折`、`跳过`。玩家选择的是“剧情往哪里走”，不是“角色下一句说什么”。

对用户来说，直接收益是：每轮都有更明确的剧情推进方向，可以更快进入关系变化、信息揭露、冲突升级、第二天或下一个剧情节点，避免长期困在同一个场景里打转。

### 8. 不用手动寻找和调试预设

酒馆式文游常常依赖外部预设。不同题材、不同人物关系、不同剧情阶段需要不同写法，用户要自己找预设、导入预设、调参数、改风格规则，才能让故事味道接近目标。

Text Game Agent 把这些预设能力内置进 harness。初始化时，LLM 会根据故事资料自动生成导演风格、叙事风格和开局配置；运行时，Director 和 Narrator 会根据当前场景、人物关系和剧情压力自动调整文风、节奏和对白密度。

对用户来说，直接收益是：不用考虑“这个故事该配什么预设”。小说、人物卡、世界书或零散设定导入后，系统会自动决定更适合的文风和剧情风格，并在不同场景之间动态切换。如果对自动生成的风格不满意，也可以用自然语言强行介入，要求 AI 按自己的想法修改导演风格、叙事风格或对白风格。

### 9. 等待时间更短

酒馆式长提示词会把设定理解、历史回忆、剧情规划、正文写作、状态整理和自我修正都压进一次调用。模型要先处理很长的提示词，再一次性完成所有任务，用户经常要等很久才看到正文。

Text Game Agent 把一轮交互拆成多个任务模块，正文生成完成后先流式显示给用户，后续总结、状态更新和召回可以在后台继续运行。

对用户来说，直接收益是：体感等待时间更短。以 DeepSeek V4 Flash 为例，一轮交互通常约 15 秒就能看到本轮正文，而不是等整套长提示词任务全部跑完。

## 基本流程

```text
玩家输入或选择剧情方向
  -> Director 判断本轮推进和场景过渡
  -> Narrator 输出小说正文
  -> OptionStrategist 生成下一轮剧情方向
  -> 玩家继续选择或自由输入

后台：
  -> Summary 更新人物状态、记忆和历史总结
  -> RecallWorker 预取可能需要的旧正文
```

## 运行

```bash

npm run web
```


## 当前目标

Text Game Agent 的目标不是替代人类小说设计，也不是模拟传统游戏的即时操控感。它更适合处理传统游戏难以细腻表达的部分：人物关系变化、对话压力、长期情绪后果、信息误判和剧情方向选择。

它希望把 LLM 文游从“长提示词聊天”推进到“有程序骨架的互动小说系统”：正文由模型生成，长期状态由程序维护，剧情方向由高层候选项控制。
